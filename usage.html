Usage

the best way is to look on demo site where you can try how it works

Business Overview

it show all features and scenarios - when you enter demo site it has redirection that aggregate scenarios by milestone

you can review or edit feature content by clicking on feature or scenario

on the right there is filter panel that allow scope scenarios by tag

  • @mc @m8
    means that i want to see all scenarios that are assigned to developer mc nad milestone m8 
  • @mc @_todo,@_spec ~@m0
    all scenarios for developer mc with status todo or spec but without assigned to milestone 0 (backlog)

we can also aggregate by two dimension, i very often use status and bucket combination

last option is adding new feature file ( you can write feature name like this: Knowledge Base - Pro Categories and the filename would be Knowledge_Base___Pro_Categories.feature

Statistics

the main goal of this view is to have common sense of how the project is going on, it can also be scoped by tags as same as in business overview. the most common is scoping by current milestone

in future we would add all milestones list - including backlog (@m0) and future milestones
